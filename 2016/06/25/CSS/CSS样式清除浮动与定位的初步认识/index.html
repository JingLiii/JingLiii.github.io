CSS样式清除浮动与定位的初步认识

2016-06-25

清除浮动

浮动不再占用原文档流的位置,会对后面的元素产生影响.

  • 浮动的本质:
  • 父盒子中的子元素浮动后,若是没有给定宽高,宽高没有办法被浮动的盒子撑起来.

    清除浮动的方法

    将浮动的盒子圈在里面,把父盒子的出入口进行闭合.利用了浮动的盒子不会跑到块级元素上面的这个特性.

    清除浮动的属性

    clear
  • left: 清除左侧浮动
  • right: 清除右侧浮动
  • botn: 清除两侧浮动

    额外标签法

    在浮动元素的末尾添加一个空标签,任何类型的标签都可以

父级元素添加overflow属性

通过触发BFC的方式,可以实现清除浮动的效果.

BFC

(Block Formatting Content)全称是块级格式化上下文,用于对块级元素的排版,默认情况下只有根元素body一个块级上下文,但是如果元素添加了overflow: hidden,position: absolutely这样属性后,就会变成BFC,独立的上下文块级元素可以包括浮动盒子.

  • 可以触发BFC机制的有:
    • folat: 不为none;
    • overflow: 不为visible
    • display: 值为inile-block,table-cell,table-caption
    • position: 为absolute 或者 fixed
    • display: table也可以触发,因为会生成一个匿名的table-cell
  • overflow: hidden||auto||scroll 都可以实现.

    使用after: 伪元素清除浮动

  • content后面的内容尽量不要为空,在火狐7.0以前的版本中会生成空格.

    after伪元素空余字符法

    content: “\200B”可以生成一个零宽度的空格,即U+200B.代替.,而不用再使用visibilty: hidden.

使用after,before双伪元素清除浮动

  • 使用 zoom:1触发 hasLayout。

    定位

    定位的属性

  • 偏移量
  • top,bottom,left,right
  • 定位模式
  • static(默认),absolute,relative,fixed

  • 静态定位 static

    其实就是默认值,就是文档流,就是啥都没有

    相对定位 relative

    保持原来的位置不变,与原来的位置进行对比,从而形成定位

    相对定位 absolute

    完全脱标,不占位置,移动的距离通过上一层中带有定位的元素进行确定
    如果没有给移动的位置,就会按照标准流来进行显示,但是不占位置

    子绝父相

    固定定位 fixed

    不管浏览器的窗口任何改变,滚动也好,放大缩小也好,都会按照与浏览器的偏移量进行定位

    定位总结

    叠放次序

    当有多个元素进行定位时,可以使用此属性进行叠放

  • z-index 取值可以为正整数,负整数,0
  • 默认值为0,取值越大越考上.
  • 取值后面不加数字
  • 只有绝对定位,相对定位,固定定位有此属性,其他的文档流,浮动都没有这个属性
    元素定位后也会进行模式转换,绝对定位与固定定位转化为行内块元素

    元素的显示和隐藏

    display

  • none: 隐藏,完成不占位置
  • block: 转化为块级元素,也就是显示

    visibility

  • visbile: 对象可视
  • hidden: 对象隐藏,隐藏后保留原有位置

    overflow

  • visbile: 不剪切内容,也不添加滚动条
  • auto: 自动添加滚动条
  • hidden: 溢出隐藏
  • scroll: 不管如何,都显示一个滚动条
